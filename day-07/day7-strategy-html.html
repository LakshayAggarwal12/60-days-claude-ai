Effort Level Guide
Low
Definitions, one-liners, quick fixes. "What does useEffect do?"
Standard
★ DEFAULT
Daily default. Debugging, CRUD routes, learning, building components.
High
Complex bugs, AI feature design, system flow, multi-step problems.
Max
Architecture reviews, capstone projects, deep AI pipeline design.
Task → Model → Effort Lookup
Task Model Effort Reason
Quick syntax / definition question Haiku Low Simple, fast, no depth needed
Debugging a specific error Sonnet Standard Needs reasoning, not max compute
Building a MERN CRUD feature Sonnet Standard Full-stack context, balanced output
Learning a new concept (JWT, RAG) Sonnet Standard Needs clear explanation + examples
Integrating an AI API (OpenAI, Gemini) Sonnet High Multi-step, error-prone integration
Designing a Gen AI / RAG pipeline Opus High Deep architecture reasoning required
Capstone / portfolio architecture Opus Max High-stakes, complex, long output
Mistakes to avoid
  • ✕Using Opus for every task — burns usage limit
  • ✕Pasting errors without code context
  • ✕Asking one giant question at once
  • ✕Skipping Claude Code for multi-file work
  • ✕Not iterating / refining AI outputs
Key learnings — Day 7
  • Context quality changes output quality dramatically
  • Model choice = strategic decision, not default habit
  • Effort level controls depth of reasoning
  • Structured prompts outperform vague ones every time
  • Claude Code > chat for real project codebases
  • Iterate outputs — first draft is never the best draft
🏆

If I could use only one model + one effort — Sonnet 4.6 · Standard

For a daily-use student focused on MERN + Gen AI development, Sonnet at Standard is the sweet spot. Fast enough for immediate feedback, smart enough for real bugs, and capable of clear teaching. Use Opus for architecture, Haiku for throwaway lookups — but 80% of daily work lives right here.
